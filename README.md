Contract Verification Guide

This guide explains how to verify a token contract deployed through EVOZXA FUTURE.

Verification ensures that the source code published on the blockchain explorer matches the bytecode of your deployed token contract.

---

Verification Settings

Use the following compiler settings during verification.

Parameter| Value
Compiler Version| 0.8.34
Optimization| Enabled
Optimizer Runs| 200
EVM Version| Paris
Verification Method| Solidity Standard JSON Input

«Important: All compiler settings must exactly match the original deployment configuration. Any mismatch may cause verification to fail.»

---

Step 1 — Open Contract Page

Open the EVOZ Explorer and navigate to your deployed token contract address.

---

Step 2 — Start Verification

Click the Verify Contract button on the contract page.

---

Step 3 — Select Verification Method

Choose:

Solidity (Standard JSON Input)

---

Step 4 — Configure Compiler Settings

Use the following values:

Compiler Version : 0.8.34
Optimization     : Enabled
Optimizer Runs   : 200
EVM Version      : Paris

---

Step 5 — Upload Standard Input JSON

Upload the file:

standard-input.json

This file is included in the Verification Package provided by EVOZXA FUTURE.

If needed, the latest version can be downloaded from:

https://github.com/EVOZXLaunch/evozxa-future/raw/main/assets/standard-input.json

---

Step 6 — Submit Verification

Submit the verification request.

The explorer will compile the contract using the supplied Standard JSON Input file and compare the generated bytecode against the deployed contract.

---

Verification Success

Verification will succeed when:

- Compiler version matches
- Optimization settings match
- EVM version matches
- Standard Input JSON matches the deployed source code

Once verified, the contract source code and metadata will become publicly available on the explorer.

---

Included Verification Package

The official EVOZXA FUTURE verification package includes:

LaunchKitToken.sol
LaunchKitTypes.sol
standard-input.json

These files represent the official source package used during token deployment.

---

Support

If verification fails, ensure that the compiler settings shown above are used exactly as specified and that the correct Standard Input JSON file is uploaded.
