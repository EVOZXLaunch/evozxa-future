Contract Verification Guide

This guide explains the steps to verify token contracts that have been deployed via EVOZXa FUTURE. Verification ensures that the source code published in the blockchain explorer mathematically matches the bytecode of the deployed contract, increasing transparency and trust for your community.

1. Compiler Configuration (Required)

When verifying, you must use the following compiler settings exactly:

ParametersValue
Compiler Version0.8.34
OptimizationEnabled
Optimizer Runs200
EVM VersionParis
Verification MethodSolidity (Standard JSON Input)

2. Verification Steps

Step 1: Open EVOZ Explorer and navigate to your deployed token contract address.

Step 2: Click the "Verify Contract" button on the contract page.

Step 3: Choose "Solidity (Standard JSON Input)" as the method.

Step 4: Configure compiler settings exactly as shown in the table above.

Step 5: Upload the standard-input.json file.

Download standard-input.json

3. Verification Results

Verification is successful if the compiler settings and JSON input are valid and match the deployed bytecode. Once verified, the source code will be publicly visible on the explorer.

4. Troubleshooting

If verification fails, ensure you selected compiler version 0.8.34 and the Paris EVM version. If problems persist, contact our support team.
